# ZO-CL06 — 11-04-2026.

## refactor.

- audit the entire compiler codebase for DRY, KISS, YAGNI violations and fix the clean wins
- PLAN_DRY_KISS_YAGNI_AUDIT.md documents the 12 findings, priority-ordered

## value (zo-value).

- extract `store()` helper method — 8 near-identical `store_int`, `store_float`, `store_bool`, `store_string`, `store_type`, `store_binding`, `store_template`, `store_runtime`, `store_closure` methods collapsed from ~120 lines to ~75

## runtime (zo-runtime, zo-runtime-native, zo-runtime-web, zo-runtime-render).

- unify `RuntimeConfig` — 3 near-duplicate structs across `zo-runtime`, `zo-runtime-native`, `zo-runtime-web` replaced with a single definition in `zo-runtime-render::render`
- consumers import `RuntimeConfig` directly from `zo-runtime-render` — no cross-crate re-exports, no fake ownership of types defined elsewhere
- main `Runtime::run()` passes `self.config` directly to backend without per-backend config reconstruction

## driver (zo-driver).

- extract `read_source()` helper in `cmd.rs` — file existence check + read + error handling deduplicated from `build.rs` and `run.rs`

## codegen (zo-codegen).

- extract `with_backend()` helper — `generate()`, `generate_artifact()`, `generate_asm()` each had the same `match self.target` dispatch, collapsed via a closure-based helper

## codegen-arm (zo-codegen-arm).

- split `codegen.rs` — extract template compilation into `codegen/template.rs` submodule
- `handle_template` and `generate_ui_entry_point` moved to the new file via `pub(crate) mod template;` + `impl ARM64Gen` block
- fields `emitter`, `functions`, `string_fixups`, `template_data` marked `pub(super)` for submodule access
- constants `TEMPLATE_HEADER_SIZE`, `TEMPLATE_CMD_SIZE`, `TEMPLATE_SYMBOL_OFFSET`, `UI_ENTRY_SYMBOL` marked `pub(super)`
- `codegen.rs` shrinks from 2470 to 2282 lines
- remove empty `register.rs` stub module and its `mod register;` declaration
- remove dead `generate_handle_event` doc comment block (orphaned after the reactive ARM codegen was reverted)

## sir (zo-sir).

- refactor `offset_value_ids` — 108-line match statement replaced with a reusable `Insn::visit_value_ids_mut` method that walks every `ValueId` in any instruction
- `offset_value_ids` becomes a trivial 3-line wrapper calling the visitor
- future passes that need to rewrite value IDs can use the same visitor instead of duplicating the match

## ui-protocol (zo-ui-protocol).

- implement loader parsing for `UiCommand::TextInput` and `UiCommand::Image` — were stubbed with `TODO` and returned `None`
- add `TextInputData` and `ImageData` `#[repr(C)]` structs matching the ARM64 binary layout
- runtime can now load all 8 `UiCommand` variants from compiled binaries

## tests.

- 3 templating wip-fail tests promoted to passing after verifying they work — `00-zsx-comments-block-unterminated`, `072-zsx-nested-fragment`, `04-zsx-nested-component`
- 218 zo program tests pass (up from 215)
- executor now inlines nested component templates — `<nested />` looks up the tag name as a local template variable, finds the matching `Insn::Template` in SIR, and extends the parent template's commands with the child's commands (15-line change in `emit_opening_tag` `TagKind::Unknown` branch)
