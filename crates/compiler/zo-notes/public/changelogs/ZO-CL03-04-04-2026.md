# ZO-CL03 — 04-04-2026.

## executor (zo-executor).

- fix `Fn(T) -> R` type annotation resolving to `Unit` instead of `Ty::Fun` — the declaration type annotation loop called `resolve_type_token` for `Token::FnType`, which fell through to the default `Unit` case. now routes `FnType` to `resolve_fn_type()`
- add annotation-to-closure type propagation — when a closure has untyped params (e.g. `fn(x)`) and the enclosing `imu` has a `Fn(int) -> int` annotation, param types are unified before executing the body
- fix closure SIR emission breaking DCE — closure `FunDef` + body was emitted inline inside the enclosing function's SIR range, causing `build_function_map` to treat the nested `FunDef` as a function boundary and truncate the enclosing function to zero-body. closure instructions are now buffered in a temporary SIR and spliced before the enclosing function's `FunDef` after its `Return`
- add `name` field to `FunCtx` — param index lookup used `body_start` to find the current function in `self.funs`, causing collisions between closures and standard library functions sharing the same `body_start`. now matches by function `name`
- fix param index resolution for multi-param closures — `self.funs.iter().find(|f| f.body_start == ctx.body_start)` matched the wrong function for closures, causing all params to resolve to index 0. changed to `f.name == ctx.name`
- add `deferred_closures: Vec<Insn>` field for buffering closure SIR during execution

## codegen (zo-codegen-arm).

- add forward-reference call fixups — closures appear before the calling function in the SIR after hoisting, but the codegen processes instructions linearly. calls to not-yet-registered functions now emit a placeholder `BL` and record a `call_fixups` entry, patched after all instructions are translated

## emitter (zo-emitter-arm).

- add `patch_bl` method to overwrite a previously emitted `BL` instruction at a given code offset with a new target offset
