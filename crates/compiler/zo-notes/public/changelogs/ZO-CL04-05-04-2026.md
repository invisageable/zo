# ZO-CL04 — 05-04-2026.

## executor (zo-executor).

- fix closure calling a captured closure — captured closure variables now copy the `ClosureValue` instead of `store_runtime`, so `resolve_closure_call` works inside closure bodies
- fix closure value leaking onto type stack in expressions — closure Ident handler now checks if the closure is being called (next token is LParen) vs passed as argument, only skipping in the call case
- fix by-copy capture semantics — capture SIR values are stored at closure creation time and reused at call sites instead of emitting a fresh `Load` that reads the current (possibly mutated) value
- fix constant folding across executor passes — `execute_binop` now guards `fold_binop` with `lhs_is_const && rhs_is_const` checks on `Value` kind to prevent incorrect folding when one operand is `Value::Runtime`
- fix function call as RHS of binary operator (`1 + f(x)`) — added `resolve_call_target` helper that scans past one operator to find function/closure Ident with semantic validation against `self.funs`/closures. added `pending_call_rparen` to suppress deferred binops during call arg evaluation
- add `Fn(T) -> R` as function parameter type — executor resolves `Token::FnType` via `resolve_fn_type` in function param parsing. unresolved Fn-typed param calls now emit `Call` with proper return type from the `Fn` annotation instead of defaulting to unit
- add closure-param monomorphization — at call sites where a closure is passed to a Fn-typed param, the executor creates a specialized copy of the function body with `Call { name: param }` replaced by the concrete closure function name. reuses existing monomorphization infrastructure
- add `FunCtx.name` field for param index resolution — replaces `body_start` matching which collided between closures and stdlib functions sharing the same value

## parser (zo-parser).

- add `Token::FnType` handling in `TypeAnnotation` state — buffers `FnType` into `expr_buffer` so the executor can see it in the tree for function parameter type annotations like `f: Fn(int) -> int`

## codegen (zo-codegen-arm).

- fix float param spill in function prologue — detect float params by `TyId` and use `STR Dn` (FP store) instead of `STR Xn` (GP store)
- fix float param reload in `LoadSource::Param` — use `LDR Dn, [SP, #off]` from spill slot instead of `FMOV Dn, D_idx` from (clobbered) original register
- fix float return in `Insn::Return` — detect float return type and use `FMOV D0, Dn` instead of `MOV X0, Xn`
- fix register clobbering in call arg setup — pre-save conflicting source registers to X16 before sequential `MOV` when a later arg's source overlaps an earlier arg's destination
- add forward-reference `call_fixups` — closures hoisted before main emit placeholder `BL` instructions, patched after all functions are registered

## emitter (zo-emitter-arm).

- add `patch_bl` method for forward-reference BL fixup patching
