# ZO-CL01 — 01-04-2026.

## codegen (zo-codegen-arm).

- add `BinOp::BitXor` emission via `EOR` instruction. previously `^` and `^=` silently did nothing — the `emit_eor` method existed in the emitter but was never wired up. removed unreachable `_ => {}` catch-all that masked the missing arm.
- add `Insn::UnOp` handler for unary negation (`NEG` via `SUB Xd, XZR, Xn`) and logical not (`EOR` with 1).
- fix signed integer display in `emit_itoa_and_write`. negative values printed as large unsigned numbers because the itoa routine used `UDIV`. now detects sign bit, emits `-` prefix, and negates before the digit loop.

## parser (zo-parser).

- add `Token::UnaryMinus` to distinguish unary negation from binary subtraction at the token level. the parser sets it via `is_unary_context()`; the executor dispatches unambiguously.
- fix unary operator postfix ordering. `handle_unary_operator` no longer pushes to `expr_buffer` directly. unary ops are stashed in `unary_spans` and drained right after the next operand in `handle_operand`, producing correct postfix order without interfering with the shunting-yard.
- fix `is_unary_context()` returning true after `)` when `expr_buffer` is empty. added `last_was_value` flag set by `handle_rparen_closer` to prevent binary `*`/`&` from being misidentified as unary dereference/reference.
- fix `handle_lbracket` array indexing branch only matching `ParserState::Expression`. inside function call parens (state `Block`), `v[i]` fell through to the array literal branch, producing wrong postfix order. now matches `Expression | Block`.

## executor (zo-executor).

- simplify `Token::Minus` dispatch to always call `execute_binop(Sub)`. unary negation is now handled exclusively by `Token::UnaryMinus => execute_unop(Neg)`.

## tests.

- add `xor-operation.zo` — xor accumulation in a loop (4 assertions).
- add `remove-duplicates.zo` — in-place dedup of sorted array (7 assertions).
- add `reverse-integer.zo` — digit reversal with sign handling and overflow detection (7 assertions).
- add `zo-executor/src/tests/bitwise.rs` — 7 unit tests for `^`, `&`, `|` binops and `^=`, `&=`, `|=` compound assignments at SiR level.
