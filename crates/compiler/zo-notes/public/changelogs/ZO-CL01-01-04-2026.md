# ZO-CL01 ‚Äî 01-04-2026.

## parser (zo-parser).

- add `Token::UnaryMinus` to distinguish unary negation from binary subtraction at the token level. the parser sets it via `is_unary_context()`; the executor dispatches unambiguously.
- fix unary operator postfix ordering. `handle_unary_operator` no longer pushes to `expr_buffer` directly. unary ops are stashed in `unary_spans` and drained right after the next operand in `handle_operand`, producing correct postfix order without interfering with the shunting-yard.
- fix `is_unary_context()` returning true after `)` when `expr_buffer` is empty. added `last_was_value` flag set by `handle_rparen_closer` to prevent binary `*`/`&` from being misidentified as unary dereference/reference.
- fix `handle_lbracket` array indexing branch only matching `ParserState::Expression`. inside function call parens (state `Block`), `v[i]` fell through to the array literal branch, producing wrong postfix order. now matches `Expression | Block`.
- fix unary `!`/`-` on function call and array index results. `!f(x)` emitted `!` between the function name and `(`, breaking the call. `handle_operand` now skips `unary_spans` drain when the next token is `(` or `[`. `unary_spans` are saved/restored across call and index boundaries (`saved_unary_spans`), and drained after `)` or `]` so `!` applies to the complete result.

## SiR (zo-sir).

- add `Insn::Nop` ‚Äî dead instruction that occupies a slot without producing a value. used by constant folding to replace operand instructions in-place so indices stay stable for codegen and register allocation.

## executor (zo-executor).

- simplify `Token::Minus` dispatch to always call `execute_binop(Sub)`. unary negation is now handled exclusively by `Token::UnaryMinus => execute_unop(Neg)`.
- fix constant folding leaving dead operand instructions and annotations. when a binop folds (e.g. `1 + 2` ‚Üí `3`), the two operand `ConstInt` instructions are now replaced with `Nop` in-place and their annotations are removed. previously `1 + 2` produced 3 SiR instructions and 3 annotations; now produces 1 live instruction and 1 annotation.
- fix array literal element type hardcoded to `int`. infer `elem_ty` from the first element's type on the type stack. arrays now work for `str`, `bool`, `float`, and any future type.
- fix array index element type hardcoded to `int`. resolve `elem_ty` from the array's `ArrayTy.elem_ty` via the type table.
- array literals now carry `size: Some(count)` for static array size validation.
- fix chained array indexing (`grid[0][0]`). `LBracket` handler now detects indexing when previous token is `RBracket`, not only `Ident`.
- extract `resolve_array_type` helper for N-dimensional array type annotations. replaces 4 duplicated single-level parsing sites. handles any depth (`[N][M][K]T`) via dimension-collecting loop + inside-out construction.

## type checker (zo-ty-checker).

- fix array size unification rejecting dynamic-to-static assignment. `[]T` (size `None`) is now compatible with `[N]T` (size `Some(N)`). only rejects when both sides are fixed and differ.

## codegen (zo-codegen-arm).

- add `BinOp::BitXor` emission via `EOR` instruction. previously `^` and `^=` silently did nothing ‚Äî the `emit_eor` method existed in the emitter but was never wired up. removed unreachable `_ => {}` catch-all that masked the missing arm.
- add `Insn::UnOp` handler for unary negation (`NEG` via `SUB Xd, XZR, Xn`) and logical not (`EOR` with 1).
- fix signed integer display in `emit_itoa_and_write`. negative values printed as large unsigned numbers because the itoa routine used `UDIV`. now detects sign bit, emits `-` prefix, and negates before the digit loop.
- `is_string_value` and `is_float_value` now recognize `ArrayIndex` results. previously `showln(arr[i])` printed raw pointers for string arrays and raw bits for float arrays.
- `ArrayLiteral` codegen: float elements stored via `emit_str_fp` instead of `emit_str`.
- `ArrayIndex` codegen: float elements loaded via `emit_ldr_fp` into FP registers.
- `ArrayStore` codegen: float elements stored via `emit_str_fp`.

## register allocation (zo-register-allocation).

- `insn_is_fp` now recognizes `ArrayIndex` with float `ty_id`. float array elements are allocated to FP registers instead of GP.
