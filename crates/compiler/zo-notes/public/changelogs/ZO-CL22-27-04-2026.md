# ZO-CL22 — 2026-04-27

## `while cond => expr;` — line form parity with `for`.

```zo
mut i: int = 0;
while i < 5 => i += 1;
showln(i);  // expected: 5
```

printed `1`. Same shape as the recently-fixed
`loop { ... }` bug: the line form parsed but never
looped. `execute_while` only emitted a label and
pushed a branch ctx — the main walk drives the cond
+ LBrace `BranchIfNot` + RBrace close for the **block**
form, but there's no `=>`/Semicolon hook in the main
walk, so the line form ran the body once and fell
through with no back-edge.

### approach.

Mirror `execute_for`'s line-form inline finalize. The
block form keeps the existing main-walk-driven path
(works fine, identical for both for and while).
`execute_while` now scans for `LBrace` vs `FatArrow` at
header depth 0 up front; if it finds a `FatArrow`, it
inline-finalizes:

```text
loop_label:
  <sub-walk cond from idx+1..fat_arrow_idx>
  cond_sir = pop sir_values
  BranchIfNot cond_sir, end_label
  push_scope
  <sub-walk body from fat_arrow_idx+1..=semicolon_idx>
  drain leftover stack
  Jump loop_label
end_label:
  pop_scope
  branch_stack.pop
  skip_until = semicolon_idx + 1
```

The block form is unchanged: scan returns `None`, the
function pushes the branch ctx and returns; the main
walk continues with the cond expression and the LBrace
fires the existing branch-emission path.

### executor.

`zo-executor/src/executor.rs` —
`execute_while(start_idx, end_idx)` now takes the real
indices (was `_start_idx`, `_end_idx`). New
`scan_while_body_marker` walks the header at depth 0,
returns `Some((fat_arrow_idx, semicolon_idx))` for line
form or `None` for block form. Paren/bracket depth
tracking matches the for-loop scanner so a `=>` inside
e.g. `f(a => b)` (hypothetical) wouldn't be mistaken
for the loop-body marker.

### regression.

`programming/99-bottles-of-beer-on-the-wall.zo` golden
re-baked to its full 397-line output (was an `EXPECTED
OUTPUT: DONE` stub from the WIP move). The program
itself uses the **block** form `while count > 0 { ... }`
and was always correct — only the test's golden was
incomplete. Test now exercises the full song through
shipping.

### suite.

clippy clean.
