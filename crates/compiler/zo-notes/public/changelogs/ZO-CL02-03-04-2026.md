# ZO-CL02 — 03-04-2026.

## executor (zo-executor).

- fix `#dom` directive not launching native renderer. the `Semicolon` token inside the directive's children was processed by `execute_node`, which finalized the expression and popped the template value off the stack. the `"dom"` match then saw an empty stack and skipped `Insn::Directive` emission. fix: skip `Semicolon` tokens when processing directive argument children.
- remove debug `eprintln` for `PRE-FRAG` / `POST-FRAG` in template fragment handling.

## module system.

- revert module system v2 (shared interner, `ModuleCompiler`, selective imports, `pub pack`/`pub load`, `PackDecl` removal). the shared interner approach caused symbol ID collisions across module boundaries and required ~60 file changes that introduced cascading regressions. the committed module system (preload with hardcoded array, `extract_exports` with dual interner translation) remains stable. user multi-file projects (`pack utils; load utils;`) are blocked until the shared interner refactor is done properly in a dedicated session.

## codegen (zo-codegen-arm).

- feat: booleans now print `true`/`false` instead of `1`/`0` in `showln`. added `is_bool_value` type detection and `emit_bool_and_write` — uses `CBZ` branching over two `ADR`-resolved static strings ("true" / "false") with shared syscall write merge point. wired into `emit_typed_write` before the int fallthrough path.

## how-zo.

- update `005-arrays.zo` — added static arrays (`[3]int`, `[2]str`), 2D arrays (`[2][3]int`), 3D arrays (`[2][2][2]int`), and all-types showcase (`bool`, `float`, `str`). updated documentation section with full type reference for dynamic, static, and N-dimensional arrays.
- update expected output in `002-primitives`, `011-binops`, `022-structs`, `024-apply`, `036-functions`, `040-inference` — `1`/`0` → `true`/`false` for bool values.
