## feature — module system with `pack`/`load` gate.

Modules now require declaration in `lib.zo` via `pub pack`.
`load foo;` looks up the module table (populated from lib.zo
packs) before falling back to filesystem resolution. This
matches Rust's `mod`/`use` two-phase model.

### how it works.

```
myproject/
  lib.zo         # pub pack mylib;
  main.zo        # load mylib;
  mylib.zo       # pub fun add(a: int, b: int) -> int { ... }
```

1. Compiler discovers `lib.zo` next to the input file
2. Scans `lib.zo` for `pub pack` declarations
3. For each pack: compiles the `.zo` file, extracts exports,
   stores in `module_table: HashMap<Symbol, ModuleExports>`
4. User `load` statements check `module_table` first
5. If lib.zo exists but module not declared → error:
   "module `foo` not declared — add `pub pack foo;` to lib.zo"
6. If no lib.zo → filesystem fallback (single-file projects)

## feature — `?` operator for error propagation.

`expr?` unwraps `Result::Ok(v)` to `v` or propagates
`Result::Err(e)` via early return. Same for `Option`.
Follows Rust's `?` semantics.

```zo
fun compute(x: int) -> Result<int, int> {
  imu val: int = might_fail(x)?;
  return Result::Ok(val + 1);
}
```

### executor (zo-executor).

- `Token::Question` handler: when not in ternary context,
  calls `execute_try_operator`
- `execute_try_operator`: loads discriminant via `TupleIndex`,
  compares against Ok (0), branches to error path if Err,
  Ok path extracts payload via `TupleIndex { index: 1 }`,
  Err path returns the original value via `Return`
- no new SIR instructions — reuses existing BranchIfNot,
  TupleIndex, Return, BinOp

### feature — heap-backed dynamic arrays.

Empty `[]` arrays allocate on the heap via `_malloc`. Push
calls `_realloc` when capacity exceeded. Literal arrays stay
on the stack.

- rename `libm_*` → `extern_*` (external C function infra)
- `emit_extern_call`: save/restore caller-save registers
- `ArrayLiteral []`: malloc heap block, initial cap 1024
- `ArrayPush`: realloc path with cap doubling + pointer
  writeback to mutable slot
- register allocator: `ArrayLiteral []` and `ArrayPush` set
  `has_calls = true` for caller-save reserve
- frame size: empty arrays use 1 stack slot (pointer) instead
  of 528 bytes
