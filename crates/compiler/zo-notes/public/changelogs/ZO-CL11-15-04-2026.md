# ZO-CL11 — 15-04-2026.

## bugfix — block arms in match consume outer while context.

`_ => {}` inside a `while` loop's `match` broke the loop:
the closing `}` of the block arm consumed the outer `while`'s
`BranchCtx`, emitting the back-edge jump inside the wildcard
arm and leaving no back-edge for the actual while body. The
loop executed exactly once.

### root cause.

`RBrace` handler checked `branch_stack.last()` unconditionally.
Inner blocks (match arm `{}`) at deeper scope depths found and
consumed branch contexts belonging to outer control flow.

### fix — scope depth guard on `BranchCtx` (zo-executor).

- add `scope_depth: usize` to `BranchCtx`, set at creation
- `RBrace` only closes a control flow block when
  `scope_stack.len() == ctx.scope_depth + 1` (same pattern
  already used for function body closing)
- inner blocks at deeper scope depths pass through harmlessly
- also added `apply_deferred_binop()` before match arm
  finalization — match arms use commas not semicolons, so
  deferred binops were never flushed for expressions like
  `tape[0] = tape[0] + 1`

### files changed.

| file | change |
|------|--------|
| `zo-executor/src/executor.rs` | `scope_depth` in `BranchCtx`, guard in RBrace, deferred binop flush in match arms |

### verified with.

- `brainfuck.zo` — full interpreter: `5, 6, 72, done`
- `match-array-test.zo` — isolated reproducer: `tape[0]`
  accumulates to `5` across 5 while iterations
- 247/247 zo_test pass, zero regressions

## tests — unit tests for match primitives and scope depth.

Added 11 new unit tests in `zo-executor` covering match on
all primitive types and the scope depth regression. Fixed 2
pre-existing test failures.

### new tests (zo-executor/src/tests/matching.rs).

- `match_char_literal_emits_cmp_chain` — char pattern dispatch
- `match_bool_literal_emits_cmp` — bool true/false patterns
- `match_bytes_literal_emits_cmp` — bytes pattern dispatch
- `match_float_literal_emits_cmp` — float pattern with ConstFloat
- `match_arm_variable_assignment_emits_store` — `ptr = ptr + 1`
  in match arm flushes deferred binop and emits Store
- `match_arm_array_store_emits_array_store` — `tape[0] = tape[0] + 1`
  emits ArrayIndex + BinOp + ArrayStore
- `match_block_arm_inside_while_preserves_loop` — `_ => {}` does
  not consume outer while BranchCtx (scope depth regression)
- `match_block_arm_inside_while_array_accumulates` — full
  brainfuck pattern: array store in match arm inside while
- `match_if_inside_block_arm_in_while` — nested if inside match
  block arm inside while (triple nesting)

### new tests (zo-executor/src/tests/arrays.rs).
