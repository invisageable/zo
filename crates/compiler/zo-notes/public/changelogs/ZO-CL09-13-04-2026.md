# ZO-CL09 — 13-04-2026.

## feature — string indexing (`s[0]` returns `char`).

`s[n]` now returns the character at byte position `n`. ASCII fast
path — single `LDRB` instruction, O(1), zero allocation. Bounds
checked at runtime (exit 1 on out-of-bounds). Char literals now
print as characters (`showln('z')` prints `z`, not `122`).

### syntax.

```zo
imu s: str = "hello";
imu first: char = s[0];     -- 'h'

showln(s[0]);                -- h
showln(s[4]);                -- o

check@eq(s[0], 'h');         -- char comparison
check@eq(s[4], 'o');
```

### error handling (zo-error).

- add `InvalidIndex` error variant — reported when indexing a
  non-indexable type or using a non-integer index
- appended to end of `ErrorKind` enum to preserve existing error
  codes

### executor (zo-executor).

- `RBracket` handler: when base type resolves to `Ty::Str`, set
  result type to `char_type()` instead of falling through to
  `int_ty`
- validate index type is `Ty::Int { .. }`, report `InvalidIndex`
  otherwise
- report `InvalidIndex` when indexing a non-array, non-string type

### emitter (zo-emitter-arm).

- add `COND_CC` constant (0x3 — carry clear / unsigned <)
- add `emit_bcc()` — branch if carry clear (B.LO), used for
  bounds checking

### codegen (zo-codegen-arm).

- add `CHAR_TYPE_ID` (3) and `STR_TYPE_ID` (4) constants — replace
  hardcoded `ty_id.0 == 4` in `is_string_value` and
  `emit_field_write`
- add `is_char_value()` — detects `ConstInt`, `Load`, `ArrayIndex`
  with char type
- add `emit_char_and_write()` — writes a single byte to fd via
  `SYS_WRITE`, same technique as `emit_newline`
- `ArrayIndex` handler: when `ty_id == CHAR_TYPE_ID`, take the
  string path — bounds check (`CMP` + `B.CC` + exit) then `LDRB`
  at `base + 8 + index` instead of array path (`LDR` at
  `base + 8 + index * 8`)
- `emit_typed_write`: char values routed to `emit_char_and_write`
  instead of `emit_itoa_and_write`
- `emit_field_write`: char fields routed to `emit_char_and_write`

### how-to (zo-how-zo).

- `007-strings.zo` — add string indexing examples (`greeting[0]`,
  `greeting[4]`), `showln` of indexed chars, `check@eq` with char
  literals
- `002-primitives.zo` — update expected output: `showln('z')` now
  prints `z` (was `122`)

### tests.

- `str-index.zo` — new build-pass test: literal index, variable
  index, `showln`, `check@eq` with char comparison
- `test_string_index_produces_char_type` — executor unit test:
  verifies `ArrayIndex` ty_id is `TyId(3)` (char)
- `test_string_index_emits_ldrb` — codegen unit test: verifies
  `LDRB` instruction is emitted for string indexing
- `test_string_index_check_eq_char` — codegen unit test: verifies
  `BinOp::Eq` with char operands generates valid code

## feature — bytes literals (`` `z` `` returns raw byte value).

`` `z` `` now evaluates to the raw byte value (122). Printed as
integer via `showln`, not as a character. Reuses `ConstInt` SIR
with `bytes_type()` — zero codegen changes needed.

### syntax.

```zo
imu b: bytes = `z`;
showln(b);                   -- 122
showln(`A`);                 -- 65
check@eq(`z`, `z`);          -- byte comparison
```

### token (zo-token).

- change `bytes_literals` from `Vec<(u32, u16)>` (source span)
  to `Vec<u8>` (byte value)
- replace `push_bytes_span(start, len)` with `push_bytes(u8)`

### tokenizer (zo-tokenizer).

- `scan_bytes()`: extract first byte between backticks as `u8`
  value, store via `push_bytes(byte_val)`

### parser (zo-parser).

- `Token::Bytes` case in `extract_value()`: changed from
  `TextRange(start, len)` to `Literal(literal_index)`, collapsed
  into the existing `Int | Float | Char` arm

### ty-checker (zo-ty-checker).

- add `bytes_type()` helper method
- add `(Ty::Bytes, Ty::Bytes)` arm in `unify()`
- add `"bytes"` to `resolve_ty_symbol()`

### executor (zo-executor).

- add `Token::Bytes` handler: reads from `bytes_literals`, emits
  `ConstInt` with `bytes_type()` — identical pattern to
  `Token::Char`
- update `BytesType` keyword to use `bytes_type()` helper

### codegen (zo-codegen-arm).

- zero changes — bytes values fall through to `emit_itoa_and_write`
  (not char, not float, not bool, not string, not enum → itoa)

### tests.

- `test_bytes_literal` — executor unit test: verifies `ConstInt`
  with `ty_id=5` (bytes) and `value=122`
- `lit-bytes.zo` — integration test: `showln`, `check@eq`
- `002-primitives.zo` — uncommented `` showln(`z`) ``, added `122`
  to expected output

## feature — array `.len` property.

`arr.len` returns the array length as `int`. The `Insn::ArrayLen`
SIR instruction and ARM64 codegen already existed but were never
emitted. This change wires the executor's Dot handler to emit
`ArrayLen` when the receiver is `Ty::Array` and the member is
`len`.

### executor (zo-executor).

- add `Ty::Array` case in Dot handler — when member name is
  `len`, emit `Insn::ArrayLen` with `int_type()` and early return
- report `InvalidFieldAccess` for any other property on arrays

### tests.

  `ArrayLen` with `ty_id=8` (int)
- `array-len.zo` — integration test: `check@eq(arr.len, 5)`,

## feature — array capacity model + `.push()` method.

Arrays now have a `[len:8][cap:8][data...]` memory layout (was
`[len:8][data...]`). Empty arrays `[]` get a default capacity of
64. `arr.push(value)` appends to the array, incrementing len and
bounds-checking against cap.

### memory layout change (zo-codegen-arm, zo-register-allocation).

- `ARRAY_HEADER_SIZE` from 8 → 16 (len + cap)
- `ArrayLiteral`: stores cap at offset 8. Literal arrays: cap=len
  (tight). Empty arrays: cap=64 (default for push)
- `ArrayIndex` / `ArrayStore`: element offset from `8+i*8` → `16+i*8`
  (automatic via `ARRAY_HEADER_SIZE` constant)
- Frame size calculation updated: `2 + cap` slots per array

### `.push()` method (zo-sir, zo-executor, zo-codegen-arm).

- new `Insn::ArrayPush { array, value, ty_id }` SIR instruction
- executor: `is_dot_method_call` extended for `Ty::Array` + `push`
- executor: `execute_array_push()` pops value + receiver, emits
  `ArrayPush`
- codegen: load len/cap → bounds check (B.CC) → store at
  `data[len]` → increment len
- DCE: `ArrayPush` marked as impure (side effect)
- pretty-printer: `push %arr, %val` format

## feature — array `.pop()` method.

`arr.pop()` removes and returns the last element, decrementing
len. Panics on empty array (exit 1).

### sir (zo-sir).

- new `Insn::ArrayPop { dst, array, ty_id }` — value-producing +
  side effect (mutates len)

### executor (zo-executor).

- `is_dot_method_call`: extended for `pop` on `Ty::Array`
- `execute_array_pop()`: pops receiver, resolves element type from
  `Ty::Array(aid)`, emits `ArrayPop`, pushes result

### codegen (zo-codegen-arm).

- `ArrayPop` handler: load len → CMP #0 → B.NE (skip panic) →
  decrement len → load `data[new_len]` into dst register
- panic path: "pop on empty array" → exit(1)
